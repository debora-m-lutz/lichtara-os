## Contribution Guidelines | Diretrizes de Contribuição

### Content Standards | Padrões de Conteúdo

#### English

- **Accuracy**: Ensure technical accuracy and spiritual authenticity
- **Clarity**: Write clearly and accessibly for diverse audiences
- **Respect**: Honor both scientific rigor and spiritual wisdom
- **Integration**: Seek to bridge rather than separate different knowledge domains
- **Bilingual**: Maintain content in both English and Portuguese when possible

#### Português

- **Precisão**: Garanta precisão técnica e autenticidade espiritual
- **Clareza**: Escreva de forma clara e acessível para audiências diversas
- **Respeito**: Honre tanto o rigor científico quanto a sabedoria espiritual
- **Integração**: Busque construir pontes em vez de separar diferentes domínios de conhecimento
- **Bilíngue**: Mantenha o conteúdo em inglês e português quando possível

### Technical Guidelines | Diretrizes Técnicas

#### English

- Follow the existing file structure and naming conventions
- Use clear, descriptive filenames and directory organization
- Include appropriate metadata and references
- Test any code or technical integrations thoroughly
- Document your changes clearly

#### Português

- Siga a estrutura de arquivos e convenções de nomenclatura existentes
- Use nomes de arquivos claros e descritivos e organização de diretórios
- Inclua metadados e referências apropriados
- Teste qualquer código ou integrações técnicas completamente
- documente suas mudanças claramente

## Submission Process | Processo de Submissão

### English

1. **Fork** the repository
2. **Create** a new branch for your contribution
3. **Make** your changes following the guidelines above
4. **Test** your changes if applicable
5. **Submit** a pull request with a clear description
6. **Engage** constructively with review feedback

### Português

1. **Faça um fork** do repositório
2. **Crie** uma nova branch para sua contribuição
3. **Faça** suas mudanças seguindo as diretrizes acima
4. **Teste** suas mudanças se aplicável
5. **Envie** um pull request com uma descrição clara
6. **Engaje** construtivamente com o feedback da revisão
