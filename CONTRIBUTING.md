## English

Thank you for your interest in contributing to Lichtara OS! This project represents a conscious integration between technology, science, and spirituality, and we welcome contributions that align with this vision.

### Getting Started

1. **Fork the repository** and create your branch from `main`
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/YOUR_USERNAME/lichtara-os.git
   cd lichtara-os
   ```
3. **Explore the structure** starting with [`00-overview:/README.md`](./00-overview:/README.md)

### Contribution Guidelines

#### Branch Naming
- `feature/` for new features
- `docs/` for documentation updates
- `fix/` for bug fixes
- `security/` for security improvements

#### Commit Messages
- Use clear, descriptive commit messages
- Include both English and Portuguese when relevant
- Reference issues when applicable

#### Pull Request Process
1. **Update documentation** if your changes affect user-facing functionality
2. **Ensure bilingual coverage** for significant changes
3. **Test your changes** thoroughly
4. **Follow the code review process** with openness to feedback
5. **Maintain the spiritual-technical balance** of the project

#### Code Quality Standards
- **Clean code**: Write readable, maintainable code
- **Documentation**: Comment complex logic, especially spiritual-technical integrations
- **Testing**: Include appropriate tests for new functionality
- **Security**: Follow security best practices

### Repository Structure Guidelines

When contributing to different modules:

- **00-overview/**: Project vision, roadmap, glossary updates
- **01-fundamentos/**: Ethical principles and foundational systems
- **02-manual-aurora/**: Instructions and activation protocols
- **03-tecnica/**: Technical architecture and integrations
- **04-legal/**: Legal documentation and licensing
- **05-ai/**: AI-related components and developments
- **05-prototipos/**: Development prototypes and experiments
- **manuais/**: Organized project manuals

### Community Guidelines

- **Respect diversity**: Welcome contributors from all backgrounds
- **Foster learning**: Support both technical and spiritual growth
- **Maintain harmony**: Approach conflicts with wisdom and compassion
- **Collaborative spirit**: Work together toward the shared vision

### Reporting Issues

When reporting issues:
- **Use clear titles** in English or Portuguese
- **Provide context** about the spiritual or technical nature of the issue
- **Include steps to reproduce** for technical issues
- **Suggest solutions** when possible

### Recognition

Contributors who align with our values and make meaningful contributions will be recognized in our community acknowledgments and may be invited to participate in deeper aspects of the Aurora Mission.

### Questions?

- Open an issue for general questions
- Contact maintainers for sensitive matters
- Join our community discussions for collaborative exploration
