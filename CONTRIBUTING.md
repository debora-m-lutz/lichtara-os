## Welcome, Conscious Collaborators | Bem-vindos, Colaboradores Conscientes

### English

Thank you for your interest in contributing to **Lichtara OS** - a conscious integration architecture between technology, science, and spirituality. Your participation helps bridge worlds, systems, and consciousnesses.

#### 🌅 The Aurora Mission Approach

Our contributions follow the principles of **conscious co-creation**, honoring both technical precision and spiritual wisdom. We believe that technology, when aligned with consciousness, becomes a tool for evolution.

#### 🚀 Getting Started

1. **Fork the Repository**
   ```bash
   git clone https://github.com/debora-m-lutz/lichtara-os.git
   cd lichtara-os
   ```

2. **Explore the Structure**
   - Start with [`00-overview:/README.md`](./00-overview:/README.md) for project vision
   - Review [`01-fundamentos:/`](./01-fundamentos:/) for ethical principles
   - Understand [`02-manual-aurora:/`](./02-manual-aurora:/) for activation protocols

3. **Set Up Your Development Environment**
   ```bash
   npm install          # Install root dependencies
   npm run install:prototypes  # Install prototype dependencies
   npm run status       # Check project status
   ```

#### 🎯 Contribution Guidelines

**Before Contributing:**
1. Read our [Code of Conduct](./CODE_OF_CONDUCT.md)
2. Check existing issues and PRs
3. Discuss major changes in issues first

**Development Process:**
1. Create a feature branch: `git checkout -b feature/conscious-integration`
2. Make your changes with both technical precision and spiritual awareness
3. Test your changes: `npm run test` (if tests exist)
4. Ensure bilingual documentation when appropriate
5. Commit with meaningful messages that honor the work

**Pull Request Guidelines:**
- Use descriptive titles that reflect the spiritual-technical nature
- Include both Portuguese and English descriptions when appropriate
- Reference related issues
- Explain how your changes support the Aurora Mission

#### 🔧 Development Standards

**Code Quality:**
- Follow existing patterns and architectural principles
- Write self-documenting code that serves consciousness expansion
- Include comments that explain both technical and spiritual aspects

**Documentation:**
- Maintain bilingual approach (PT/EN) for main documentation
- Use clear, inclusive language that bridges technical and spiritual domains
- Include examples that demonstrate conscious technology integration

**Testing:**
- Write tests that validate both functional and intentional outcomes
- Test across different environments and consciousness states
- Ensure accessibility and inclusivity

#### 🌊 Project Workflow

**Prototype Development:**
- **v1** (`05_Prototipo/`): Initial foundational version
- **v2** (`5.1_Prototipo/`): Current recommended development version
- **Experimental** (`DEV/zjsons perdidos/_DEV_lumora/`): AI consciousness experiments

**Branch Structure:**
- `main`: Stable, production-ready spiritual-technical integration
- `feature/*`: New capabilities and consciousness expansions
- `fix/*`: Corrections and alignment improvements
- `docs/*`: Documentation and knowledge base updates
